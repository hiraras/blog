# requestAnimationFrame

回调函数会在绘制执行之前执行，它是 js 提供的一个渲染中，最后修改渲染内容的机会

浏览器一个渲染帧做了什么？

## 渲染帧是什么

浏览器的理想刷新频率为 60fps，在这个频率下，肉眼不会觉得卡顿，所以浏览器在理想状态下 1 个渲染帧的时间就是 1000ms/60f ~ 16.6ms

在这个时间段中，浏览器要执行事件循环，requestAnimationFrame，更新屏幕渲染，requestIdleCallback 这些事，这一套所有的事叫做一个渲染帧

## 一个渲染帧中的执行步骤

1. 事件循环 - 每个渲染帧中会分配一个时间范围来执行事件循环，如果某个宏任务执行的很快则顺序为(宏任务->清空微任务->另一个宏任务->再次清空微任务)，直到检测到分配的时间结束了，则结束本轮事件循环
2. requestAnimationFrame 回调
3. 渲染更新（包括布局/重绘/合成）
4. 如果一帧内还有空闲时间则执行 requestIdleCallback 的回调

**注意：**上面的 123 步骤在一帧中必定会执行，如果 1/2 两个步骤本身事件超过了 16.6ms，也会继续执行完这一流程的所有步骤

```js
// 一帧内执行多个宏任务示例
setTimeout(() => console.log("宏任务1"));
setTimeout(() => console.log("宏任务2"));

requestAnimationFrame(() => console.log("rAF"));
// 输出：宏任务1 宏任务2 rAF
```
