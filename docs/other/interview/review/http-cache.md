# 浏览器缓存

## 强缓存 - 命中后不发请求，直接用本地缓存

代码发出请求时，浏览器发现和历史请求一致（靠缓存键判断：url+一些维度），并且缓存时间没有过期，不向服务端发出请求，直接使用历史数据/资源（network 中显示 from disk cache）。

## 协商缓存 - 会发请求问服务器“资源变了吗”

浏览器发出请求时带上缓存相关请求头，服务端判断资源是否有更新，如果版本未更新，返回 304 状态码（不带实体），浏览器直接使用历史资源；如果版本有更新，则返回 200 状态码和新的资源。

若用户强刷（Ctrl+F5）或请求带特殊指令（如 Cache-Control: no-cache），浏览器会绕过强缓存去走协商/重新请求。

## 文件 hash 和 query

-   **文件名 hash 变化**（如 `app.a1.js -> app.b2.js`）  
    对浏览器来说是新 URL，旧缓存键不匹配，通常会直接发新请求。

-   **URL 查询参数变化**（如 `img.png?t=111 -> img.png?t=222`）  
    也会被视为不同资源，一般会重新请求。

所以给请求加时间戳参数常用于“强制绕过旧缓存”。 （前提是中间层/CDN 没有做非常规缓存键归一化；默认情况下 query 变化会区分缓存。）
