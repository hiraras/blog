# 垃圾回收

垃圾回收是**在特定时机（内存不足、空闲时间等）**，通过**可达性分析**识别并回收不可达对象内存的机制。将使用过后用不到的对象的内存地址释放，不让程序占用内存无限累加

垃圾回收针对的是存储在堆内存的数据类型，而变量是存储在栈内存中的(函数结束运行后直接释放)。以下数据类型会被垃圾回收

```js
// ✅ 会被GC回收的对象
let obj = { name: "Alice" }; // 普通对象
let arr = [1, 2, 3]; // 数组
let func = () => {}; // 函数对象
let date = new Date(); // Date对象
let reg = /abc/; // 正则对象
let map = new Map(); // Map
let set = new Set(); // Set
// 函数对象也是对象
function Person() {} // 函数对象
let p = new Person(); // 实例对象
```

## 发展过程

1. 引用计数法
2. 标记清除法
3. 分代回收+增量标记

## 细节

### 引用计数法

对象每被引用一次就会对该对象计数，当扫描时发现有对象的引用次数为 0 时，就回收该对象。

-   优点：很清晰，很好理解
-   缺点：如果两个对象有循环引用的关系则永远不会被清除

### 标记清除

为了解决引用计数中循环引用的问题，诞生了标记清除的方法。该方法是从[计数]到[可达性]思维上的转变

1. 将代码内所有对象假定为可以清除的，标记为 0
2. 将从根对象开始遍历，将还有用到的对象标记为 1
3. 清除所有标记为 0 的对象

-   优点：实现比较简单
-   缺点：清除后会产生内存碎片，影响后续大对象分配效率。因此 V8 老生代会配合**标记-整理**阶段来压缩内存。

### 分代回收+增量标记(现代 V8 引擎的方法)

标记清除存在清除后内存地址空缺(空间)和执行时(时间)造成 js 线程堵塞的问题，分代回收和增量标记是对时间和空间的两个优化

-   分代回收

将内存分为新生代和老生代

新生代中有两个区域(From-Space 和 To-Space)。数据先进入 From-Space，GC 时**将所有存活对象复制到 To-Space（复制时完成压缩）**，然后清空 From-Space，最后交换角色。这样不会有内存碎片。
当新生代中的数据经历了多次垃圾回收或数据超过一定大小，就被转移到老生代。

老生代回收的时候有三个阶段，标记，清除和整理。

1. 标记：使用三色标记，标记出所有存活的对象
2. 清除: 回收不可到达对象，产生碎片
3. 整理: 当碎片化严重时，移动对象压缩空间

分代回收实际上是将数据从生命周期的角度对数据进行了分类，同时通过两个区间，减少了扫描的空间成本

-   增量标记

旧标记清除会一次性从根节点开始遍历完所有数据节点，可能会造成 js 线程堵塞。增量标记，将这整个过程拆分为更细粒度的多个标记步骤，将每个步骤安排在 js 执行的空闲时间，以达到性能的优化。

-   标记的方法

上面提到的标记使用的是三色标记法，即将数据分为三种类型:

白色: 可删除的节点
灰色: 已访问，子节点未处理
黑色: 已访问，子节点也已处理

三色标记流程：

1. 初始：所有对象白色
2. 从根开始：根对象标记为**灰色**（已访问，子节点未处理）
3. 处理灰色节点：将其子节点标记为灰色，自身变为**黑色**
4. 重复步骤 3 直到无灰色节点
5. 最终：黑色=存活，白色=可回收

-   其他优化手段

1. 并发标记: 在主线程执行 JS 的同时，后台线程进行标记
2. 空闲时间 GC: 利用浏览器空闲时间执行 GC
3. 大对象空间: 2MB 的大对象直接分配在大对象空间，避免复制成本
4. 细胞空间: 固定大小的对象（如函数、字符串）放在细胞空间，提高分配效率

-   屏障

防止增量标记过程中，JS 代码修改对象引用，导致存活对象被误标记为白色而回收。

```js
// 增量标记执行到一半时
let parent = { child: null }; // 已被标记为黑色
let newChild = { data: 1 }; // 当前是白色（尚未扫描）

// JS代码执行：黑色对象引用白色对象
parent.child = newChild; // 触发写屏障！

// 写屏障介入：
// 检测到黑色对象指向白色对象
// 立即将白色对象改为灰色，防止被回收
```
